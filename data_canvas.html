    How to use this canvas

    Code cells run real Python in your browser (pandas, numpy, matplotlib, seaborn, pandas, duckdb, and plotly are pre-installed). The last expression in a cell is displayed — DataFrames (pandas and duckdb queries) render as tables, matplotlib/pandas plots as images, and plotly figures stay interactive. Hover a cell to reveal + handles for inserting a new cell above or below it.

    Shortcuts — Shift+Enter runs a cell and moves to the next (creating one at the end); Ctrl+Enter (or Cmd+Enter) runs in place; Ctrl+Space autocompletes.

    Markdown cells — start a cell with %%markdown to write notes. Run renders it; double-click the rendered text to edit again. Headings (#, ##, ###) build the Contents panel on the left.

    Data — everything runs inside a data/ folder. Upload files into it from the Files panel (right sidebar) and read them by plain filename — pd.read_csv("sales.csv").Please note that files are not cached between sessions. Import → CSV or Excel also creates DataFrames for you automatically (Excel: one DataFrame per sheet, named filename_sheetname). Need another package? Write %pip install package-name in a cell (space-separate multiple packages; version pins like pandas==2.2.0 work). The older install("package-name") form still runs.

    Exporting data — write a file the normal way — pandas df.to_csv("results.csv"), or duckdb COPY ... TO 'results.json' — and it appears in the Files panel with a download button. Supported types: .csv, .xlsx, .json, .parquet. (Note: pandas .to_excel() needs install("openpyxl") first; for parquet, use duckdb.) These files live in the browser's in-memory filesystem only: they aren't baked into Save Copy, and Restart clears them — download what you need first.

    Saving — your work autosaves in this browser as you type. Export → Save Copy downloads this very HTML file with your cells and their outputs baked in; open the copy later and everything is restored, no re-running needed (cells you never ran come back as plain code). Export → Notebook JSON saves cells + outputs as a small file; Python script writes a .py with # %% cell markers and outputs as comments. Import → Notebook opens either saved format, and Import → Python script parses a # %%-marked .py (including this canvas's own exports) back into cells. Note: loaded data files aren't embedded — re-load your CSV/Excel after opening a saved copy.

    Performance — tick Cache packages (experimental) to store the Python runtime and packages in your browser (IndexedDB). The first load still downloads everything; every load after that restores from cache and is dramatically faster. When active, a package_cache_info entry appears in the Memory Inspector. Use Clear cache if packages ever misbehave.

    Disclaimer

    Notebooks are programs. Opening a notebook file and running its cells executes that author's Python code in your browser. The browser sandbox keeps it away from your files and operating system, but it can still make network requests, use your bandwidth and memory, and tie up this tab. Only run notebooks from people you trust — the same rule as any Jupyter notebook or downloaded script.

    Rendered content is sanitized, not guaranteed. Markdown cells and table outputs from opened files are cleaned with DOMPurify before display. Treat shared notebook files with the same caution you'd give any file from the internet.

    Don't paste secrets into cells. Autosave keeps your work (including anything you paste, like API keys or passwords) in this browser's local storage, and Save Copy / Export bake cell contents into the files you share. Assume anything typed in a cell may end up in a shared file.

    No interrupt. An infinite loop (while True:) will freeze this tab — use Restart to recover. Very large datasets can exhaust the tab's memory.

    Dependencies come from public CDNs (Pyodide, CodeMirror, plotly, and friends), so the first load needs internet access. The experimental package cache stores downloads in this browser's IndexedDB; Clear cache removes them.

    No warranty. This is an educational tool provided as-is. Verify results independently before relying on them.
